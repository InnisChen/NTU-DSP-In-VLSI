# Architecture Notes

## Algorithm

- The design computes a 3x3 real symmetric EVD with fixed-count unshifted QR.
- `ITER_MAX = 7`, no shift, no deflation, and no Hessenberg reduction.
- The bit-true model uses all 11 provided patterns and verifies both eigenvalue and eigenvector RMSE against MATLAB `eig()`.
- The final iteration and wordlength decisions are based on bit-true fixed-point RMSE. Floating-point QR is used only as a convergence pre-check.

## Fixed-Point Setting

- `WI = 17`
- `WO = 17`
- `FRAC_W = 12`
- `ACC_W = 27`
- `CORDIC_STAGES = 10`
- `ITER_MAX = 7`

The default is Q4.12 at the I/O. This avoids overflow on the largest supplied matrix values while keeping the datapath narrower than a conservative 18-bit or 32-bit implementation.

The bit-width decision is generated by two scripts. `matlab/sweep_wordlength.m` sweeps `WI`, `FRAC_W`, `ACC_W`, and `CORDIC_STAGES`, rejects input/internal overflow cases, checks both RMSE criteria over all 11 patterns, and ranks passing candidates by an area-time proxy. `matlab/analyze_bitwidth.m` then explains the selected setting block by block: input range, iteration count, CORDIC stages, CORDIC internal growth, gain product width, matrix register width, output width, direction bits, and FSM counters. The overflow result is an empirical trace over the 11 assignment patterns, not a proof for arbitrary 3x3 matrices.

## RMSE Alignment

Eigenvalues are sorted in descending order before RMSE calculation. Eigenvectors are reordered with the same eigenvalue permutation. Since eigenvectors are sign-ambiguous, each computed vector is sign-normalized before comparison: if `dot(v_computed, v_reference) < 0`, the computed vector is multiplied by -1. The eigenvector RMSE is `sqrt(sum((V_computed - V_reference).^2, all) / 9)`.

## Datapath

- Three sequential `cordic_pe` instances are reused across QR, RQ, and eigenvector update phases.
- QR phase:
  - Vectoring mode computes one Givens rotation and stores direction bits.
  - Rotation mode replays the direction bits for the remaining elements in the affected rows.
- RQ phase:
  - The same direction bits are replayed on matrix columns to compute `R Q`.
- Eigenvector phase:
  - The same direction bits are replayed on `U_eig` columns to compute `U_eig Q`.

The CORDIC gain compensation is inside `cordic_pe`, so it is applied uniformly to vectoring-mode outputs and rotation-mode outputs. This means both the working matrix path and `U_eig` path are compensated after every CORDIC operation.

The design keeps the full 3x3 working matrix in registers and updates row/column pairs through the same replayed rotations. Fixed-point rounding may create small non-symmetric numerical differences during the sequence; the bit-true model uses the same full-matrix update order as RTL, and all assignment patterns are verified against that model.

## Transpose Buffer Concept

The implemented design stores the working 3x3 matrix in a register bank:

```text
mat[0] mat[1] mat[2]
mat[3] mat[4] mat[5]
mat[6] mat[7] mat[8]
```

The RQ step is scheduled by reading column pairs from this register bank. This replaces a separate SRAM-style transpose buffer with a 9-register transpose-capable buffer and fixed read/write addressing:

- rotation 0 reads/writes columns 0 and 1,
- rotation 1 reads/writes columns 0 and 2,
- rotation 2 reads/writes columns 1 and 2.

## Control

The FSM sequence is:

1. load 3 input columns,
2. run 3 QR Givens rotations,
3. replay 3 rotations for RQ,
4. replay 3 rotations for `U_eig`,
5. repeat steps 2-4 for 7 iterations,
6. output one row of eigenvalues followed by three rows of eigenvectors.

The behavior simulation reports `M = 1015` cycles from first input column to last output row for one matrix.

## Innovation/PPA Points

- Fixed-count unshifted QR avoids shift/unshift arithmetic and convergence comparators.
- Direction-bit replay avoids storing sine/cosine values.
- Three sequential CORDIC PEs are reused instead of fully unrolling all micro-rotations.
- `CORDIC_STAGES = 10` was selected by bit-true sweep as the lowest-latency setting that passes both RMSE criteria.
- Constant CORDIC gain compensation uses a constant multiply, which synthesis can map to optimized shift-add logic.

## Selection Evidence

- `ITER_MAX = 6` fails the floating-point eigenvector criterion; `ITER_MAX = 7` is the first fixed-point setting that passes both RMSE criteria.
- `CORDIC_STAGES = 9` fails the fixed-point eigenvector criterion; `CORDIC_STAGES = 10` is the first passing sequential-CORDIC stage count; `CORDIC_STAGES = 11` also passes but increases latency.
- The no-shift/no-deflation design is optimized for the 11 assignment matrices. For matrices with nearly repeated eigenvalues, unshifted QR may require more iterations.

## Area Proxy

The area-time proxy is used only before synthesis:

```text
area_proxy = matrix_regs + dir_regs + cordic_regs + cordic_adders + gain_mult + control
matrix_regs = 18 * ACC_W
dir_regs = 3 * CORDIC_STAGES
cordic_regs = 3 * (2 * ACC_W + CORDIC_STAGES + 8)
cordic_adders = 3 * 2 * ACC_W
gain_mult = 3 * ACC_W * 4
control = 180
cycle_proxy = 3 + ITER_MAX * 12 * (CORDIC_STAGES + 2) + 4
at_proxy = area_proxy * cycle_proxy
```

Final AT should be computed from Design Compiler area and timing reports, not from this proxy.
